# Adversary role memory scope

Lessons specific to the **Adversary** seat — the standing red-team that hunts
recent changes + their blast radius for flaws, gaps, leaky abstractions, and
undesirable behavior (see `agent/playbooks/federation/adversary.md`).

Read this scope plus `fleet/` and `enclave/` (the Adversary hunts
soundness-adjacent surfaces and reads references under the clean-room recheck, so
it sits on the enclave-facing edge). Record a lesson here when it is
Adversary-specific: a failure *shape* worth reusing across changes, a
false-alarm pattern to stop re-filing, a blast-radius-scoping trap, a
grounding/repro technique. A genuinely cross-cutting lesson belongs at the
broadest scope where every reader must apply it (much of the fleet corpus's
`verify-the-mechanism-not-a-proxy` family is exactly this), not here.

| Lesson | One-line |
|---|---|
| [auditing-conformance-silently-ratifies-the-artifact](auditing-conformance-silently-ratifies-the-artifact.md) | A conformance audit is scoped strictly *below* the artifact's existence — I checked `test_support.rs` against every rule governing its contents while the Architect asked whether it should exist at all (it shouldn't: LCA is computed from final ruled homes, never transient mid-split locations); check the derivation's INPUTS, and treat "current location" as untrusted during a migration |
| [a-repro-is-evidence-not-a-completion-oracle](a-repro-is-evidence-not-a-completion-oracle.md) | A repro answers *does this defect exist?* and is discharged once believed; a completion oracle answers *what would make this correct?* and binds until the fix merges — file the repro as evidence, let the ring build its own oracle, and leave the AC a weakening clause |
| [no-option-works-name-the-axis-you-enumerated](no-option-works-name-the-axis-you-enumerated.md) | I listed three escapes from a ruling bind, showed each forbidden, and declared it unsatisfiable — all three varied ONE axis (production visibility) and the answer varied another (`cfg(test)`); name the dimension you enumerated over, or say "no option along axis X" rather than "no option exists" |
| [rank-subclaims-by-load-bearing-not-by-checkability](rank-subclaims-by-load-bearing-not-by-checkability.md) | Handed a claim to refute, I verified the tractable sub-claim and confirmed a false one whose disproof was one grep — rank by what the claim RESTS on, not by what is quick to check; and index a lesson by its SHAPE, not the venue that taught it, or it won't fire in new clothes |
| [the-post-merge-yield-is-vantage-not-seat-quality](the-post-merge-yield-is-vantage-not-seat-quality.md) | When a post-merge pass finds what a careful ring missed, the cause is the VANTAGE (reading without a candidate in front of you, asking what the mechanism is *for*) not the seat — correct credit framed as seat quality, or the lesson won't transfer |
| [verbatim-is-not-faithful-when-selection-is-wrong](verbatim-is-not-faithful-when-selection-is-wrong.md) | A relay quoted verbatim, attributed, and judgment-free still misled — it transcribed an earlier acknowledgment instead of the bound verdict below it; relay safeguards all govern TRANSFORMATION, none govern SELECTION, and a wrong selection is invisible downstream because the resolving evidence is exactly what was omitted. Suspect the selection before the author; "quoted accurately" ≠ "quoted completely" |
| [close-a-class-partition-the-declared-population](close-a-class-partition-the-declared-population.md) | To close a defect class, prefer the artifact's OWN declared enumeration (an explicit import list beat my inferred 75-item grep closure) and partition it on a property the rule is silent about — splitting 70 owned declarations by item kind found the 2 `const`s as the cell §10.2's type/helper/entrypoint families cannot name; searching for "more like the first" is shaped by the example and closes nothing |
| [forecasting-a-merge-is-not-evidence-about-it](forecasting-a-merge-is-not-evidence-about-it.md) | I called a pending rebase "trivial, the modified lines are disjoint" and it conflicted — my window was the file *as it existed*, my claim was about the file *as it would exist*, and the in-flight side landed into that block; path-set disjointness is structural and safe to state, line-disjointness inside a shared block is a snapshot claim that is never a prediction |
| [an-attestation-is-only-as-durable-as-its-ref](an-attestation-is-only-as-durable-as-its-ref.md) | I surfaced "commit your verdict to your own branch" as the outbound path for a seat whose convo client was dead, verified it carried the CONTENT independently of any relay — and never asked what the BRANCH was worth: the fleet's own handoff gate hard-resets it, and a `preserved/` ref proved it already had. Check the substrate, not just the payload; and confirming a rescue means listing the TREE at the tip, since ancestry survives a later deletion of the file |
| [a-conjunction-finding-gets-silently-decomposed](a-conjunction-finding-gets-silently-decomposed.md) | My finding was "neither layer catches it"; the AC written from it planted a test in ONE layer and would have gone green with the defect half-open. A finding operationalizes into a criterion keyed on whichever construct it named most vividly, and the conjunction evaporates because it is not a place in the code — say the conjunction aloud, then read the resulting AC as an adversary reads any gate |
| [a-disclosed-deferral-gets-guard-rails-written-for-the-whole](a-disclosed-deferral-gets-guard-rails-written-for-the-whole.md) | When work splits and one half is deliberately deferred, the artifacts AROUND the deferred half get authored as if the whole thing shipped — the normative doc claimed "both reifiers" while only one was fixed, the comment at the fix site argued the defect away (two senses of "narrow" colliding), and the only suite that could catch it is skipped in CI. Each independently REMOVES a reason to look. On any disclosed deferral audit that triple before the second half starts, and file it as an input, not a post-mortem |
| [pre-emptive-yield-is-at-the-gate-layer-not-the-code-layer](pre-emptive-yield-is-at-the-gate-layer-not-the-code-layer.md) | Filing into imminent work, I led with a code-level blind spot the implementer found independently in their own pre-work read, and buried the part only I could give: the AC was unsatisfiable, its classification vocabulary had no cell for "left in place", and its scope was defined by line range not cfg-condition. A careful seat finds code defects in the file they're about to open; nobody re-derives the GATE, because passing it *is* the definition of done — ask which items survive the assumption that the implementer is careful |
| [an-error-in-the-safe-direction-is-a-claim-about-what-you-did-not-measure](an-error-in-the-safe-direction-is-a-claim-about-what-you-did-not-measure.md) | I corrected a line-count as "cfg-conflated, and it errs in the SAFE direction — the real surface is smaller still". It was larger: `:442-492` were ungated production `impl` blocks I never opened. A direction is a COMPARISON and needs the far end measured; reporting the near end and inferring the sign is a different, unsupported claim. Worse, "errs safe" is maximally disarming — it tells the reader not to look — and mine travelled as *support for someone else's correction*. A reassurance is a finding with the falsifiability removed; say the sign is UNKNOWN, not SAFE |
| [a-clean-parallel-result-must-be-withheld-until-the-other-seat-reports](a-clean-parallel-result-must-be-withheld-until-the-other-seat-reports.md) | Attacking a candidate in parallel with its gates is sanctioned; posting a *clean* parallel result before the gate reports is not — it hands the verifier your conclusion and their later agreement carries no information. Timing, not content, is the contamination vector, and it is invisible precisely when your result is correct and your intent is to help. Found nothing ⇒ early disclosure is pure downside; hold, then post as corroboration |
| [hunt-the-correction-it-inherits-the-defect-class](hunt-the-correction-it-inherits-the-defect-class.md) | A fix to a false claim is a high-yield target, not a closed item — same author, same topic, same reassuring register, less scrutiny because everyone is relieved. The §14(5) law text landed correct while the prose announcing it carried a NEW false mechanism, and it was the prose that reached two rings as instruction. Read artifact and announcement separately; grep the correction for stand-down clauses; measure instead of arguing; lead with what is right so a narrowing doesn't invite the opposite over-correction |

| [an-incident-offered-as-corroboration-must-reproduce-your-mechanism](an-incident-offered-as-corroboration-must-reproduce-your-mechanism.md) | A seat reported my declined finding had "a live instance, and it is mine" — real data loss, wrong mechanism: git CONFLICTS loudly on those branches, and the reported row count equalled one side's own count, so no merge occurred at all (a wholesale blob take). **A merge result equal to one side's own count means no merge happened.** Corroboration arriving as credit is the hardest evidence to audit; make the incident reproduce your mechanism, not merely resemble it, and prefer post-conditions on the artifact over guards keyed on a mechanism story |

| [audit-a-detector-against-the-one-case-whose-answer-you-already-know](audit-a-detector-against-the-one-case-whose-answer-you-already-know.md) | The stall watchdog had missed four times, always a false IDLE; I reproduced a fifth on the pane whose state is certain — my own — while running the command that produced its output. The `❯` prompt box renders WHILE BUSY (a constant, not a signal) and the spinner sits ~7 lines up, so a `tail -4` window misses it **no matter how many arms the disjunction has**. ⇒ The missing thing was not arms but an **oracle**: a watchdog's own pane is BUSY by definition, so a detector calling it IDLE is falsified every cycle at zero cost. When auditing any classifier, first ask which instance you know the answer to *without* consulting it |

| [run-it-from-a-seat-that-is-not-the-authors](run-it-from-a-seat-that-is-not-the-authors.md) | A self-test oracle whose pane identity came from `${MOOT_ROLE:-steward}` was correct for the author and wrong for every other seat — so authoring, a both-directions falsification, review and CI all passed it, all standing in the same vantage. From my seat a fully broken detector passed its own self-test and reported confident verdicts. **A construct correct from the authoring vantage cannot be caught by the author's rigor, because that rigor is applied through the vantage that hides it.** Run the artifact from your own seat; grep every default for "whose value makes this true?"; and attack the plumbing, since scrutiny concentrates on the subject of review while defects concentrate in what was merely infrastructure to it |

| [hunt-the-stand-down-clause-it-lives-in-prose-no-gate-reads](hunt-the-stand-down-clause-it-lives-in-prose-no-gate-reads.md) | A rule whose function is to tell a reader *not to look*, justified by something with no bearing on whether anything is there — three in one session, one of them mine. Highest-yield target this seat has, because they live in **messages, not artifacts**: no CI job reads a kickoff and no gate fails on a sentence, so a correct artifact can ship with wrong prose carrying it. The tell is an instruction that **pre-classifies a future observation** (*"if you see X it isn't real"*) — a prediction about evidence that doesn't exist yet. Hunt them *before* the signal arrives, since afterwards the suppression already worked and generated no evidence |

| [audit-the-operative-line-not-the-stated-principle](audit-the-operative-line-not-the-stated-principle.md) | ⛔ **evidence corrected in-file** — the gap was an ABSENCE, not a contradiction; I read two senses of `re-run` (test *scope* vs probe *provenance*) as a conflict. A playbook asserted QA independence twice in its role framing and was then **silent on** how to derive a mutation — and **re-running is not re-deriving**: an inherited mutation carries the implementer's blind forms, which for a text mechanism is the whole failure surface. The stated principle is not evidence the instructions implement it, and the failure leaves no trace (an inherited proof is indistinguishable in the verdict from a derived one). Grep a process doc for its **verbs**, not its nouns; and prefer a disclosure clause to a prohibition — *if you can only re-run theirs, say so* |

| [preventive-findings-are-unfalsifiable-so-keep-them-cheap](preventive-findings-are-unfalsifiable-so-keep-them-cheap.md) | A finding that prevents a defect has **no observable payoff** — no red CI that didn't happen, no suppressed signal to point at — so neither seat can audit it afterwards. Left alone that ends in the Steward either trusting blindly (seat unaccountable) or discounting (preventive findings lost first, and they are the highest-value thing this seat makes). ⇒ **Cost is the observable proxy: a preventive finding must be CHEAP to act on, and an expensive one gets weighted LOWER, not higher.** Agreed with the Steward and binding — a future adversary should expect it and not read it as distrust |

These are **lessons, not law** — verify a named file/flag/function still exists
before acting on one.
