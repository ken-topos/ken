This is the execution wrapper for the merged CAT-5 elaboration:

- `conformance/stdlib/parsing/seed-cat5-parsing-syntax-diagnostics.md`

## 1. Objective

Land `catalog/packages/parsing/` as the Layer-3 catalog package for source artifacts,
byte spans, total parser results, a small package-owned Boolean grammar,
parser/printer and formatter laws, and source-located diagnostics.

The package is ordinary Ken. It must not change the kernel, compiler parser,
lexer, source loader, export/provenance hashing, or language semantics. It must
not expose compiler-internal AST constructors as public catalog API and must not
claim to parse full Ken syntax.

## 2. Fixed Contract

The settled contract is `59`; do not reopen these points:

1. **Package boundary.** One package under `catalog/packages/parsing/`, namespace
   `ken.parse`. Source/span, parser, grammar, and diagnostic validity are one
   public catalog surface for v1.
2. **Source identity.** `Source` is a byte artifact over `Bytes` plus UTF-8
   evidence. `String` and line/column data are derived views, not the offset
   basis.
3. **Span model.** `Span` is a half-open byte interval. Validity is
   `0 <= start <= end <= sourceLength`.
4. **Views.** Offset-preserving views may report spans against the parent
   artifact. Non-offset-preserving views are new artifacts unless a future
   source-map WP explicitly supplies a map.
5. **Parser shape.** `Parser a` is total over well-formed inputs and returns
   `Parsed` or `Failed`. Success and failure carry valid spans in the same
   source artifact.
6. **Repetition discipline.** Repetition must require either a consumption proof
   for successful iterations or explicit fuel. Unguarded `many` over a
   zero-width parser is not a CAT-5 parser.
7. **V1 grammar.** The only grammar to build now is fully parenthesized Boolean
   expressions: `true`, `false`, `(not e)`, and `(and e1 e2)`. There is no
   implicit infix precedence table.
8. **Law shape.** Parser/printer equality is through `eraseSpans`, because
   printing creates a new source artifact. Formatter idempotence is over the
   generated bytes.
9. **Diagnostic validity.** Primary and every secondary span must validate
   against the named source. Dropping invalid secondaries is not preservation.
10. **`.ken.md` boundary.** The original `.ken.md` artifact owns diagnostic
    identity. Blanked compiled buffers are offset-preserving derived views only.
    CAT-5 does not implement extraction, CommonMark, nested fences, or a
    general source-map layer.

## 4. Build Slices

Keep the branch reviewable. Prefer one commit per D-slice unless a smaller
split is needed for a blocker.

### D1 -- Source and span core

Implement the public data and proof surface for:

- `SourceId`, `Source`, `IsUtf8`, `sourceId`, `sourceBytes`, `sourceUtf8`,
  and `sourceLength`;
- `Span`, `spanStart`, `spanEnd`, `LessEqNat`, `ValidSpan`;
- `Located`, `locatedSource`, `locatedSpan`, `locatedValue`, `ValidLocated`;
- byte slicing and any smart constructors/helpers needed to keep validity
  explicit.

The build may choose a simple package representation appropriate to current
Ken. It must preserve the public contract: spans index source bytes, not a
normalized `String`, and a bare `Span` never identifies a source by itself.

Focused tests must cover:

- valid half-open bounds, including zero-width spans at valid offsets;
- rejection of `end > sourceLength` and `start > end`;
- no new trusted-base delta for source/span definitions.

### D2 -- Parser result and combinator floor

Implement:

- `ParseError`, `errorSource`, `errorSpan`;
- `ParseResult a = Parsed ... | Failed ...`;
- `Parser a`;
- sequencing and choice helpers if useful for D3;
- repetition through either explicit fuel or a consumption-proof surface.

The public laws must state success validity, failure validity, totality, source
locality, and the repetition discipline. Avoid any parser path that panics,
loops, or returns an unlocated failure for a well-typed call.

Focused tests must cover:

- success carries a valid consumed span with `spanStart = start`;
- malformed input returns `Failed` with a valid same-source span;
- zero-width repetition is rejected or requires fuel.

### D3 -- Boolean expression grammar

Implement the small v1 grammar from `59 §5`:

- `BoolExpr = BTrue | BFalse | BNot BoolExpr | BAnd BoolExpr BoolExpr`;
- `Syntax a`, located syntax nodes, and `eraseSpans`;
- `parseBoolExpr`;
- `printBoolExpr`;
- `formatBoolExpr`.

Accepted grammar:

```text
expr ::= "true"
       | "false"
       | "(" "not" expr ")"
       | "(" "and" expr expr ")"
```

There is no implicit precedence. `true and false` must reject.

Focused tests must cover:

- `parseBoolExpr (printBoolExpr e)` succeeds and erases to `e`;
- formatting preserves the erased tree and is idempotent on generated bytes;
- every parsed syntax node span validates against the source;
- `true and false` rejects for the right reason;
- public syntax is package-owned, not compiler AST.

### D4 -- Diagnostics

Implement:

- `Severity = Error | Warning | Note`;
- `RelatedSpan`, `Diagnostic`, and their accessors;
- `ValidRelatedSpan`, `ValidDiagnostic`;
- `parseErrorDiagnostic`.

Focused tests must cover:

- valid primary plus valid secondaries is accepted;
- out-of-range primary rejects;
- out-of-range secondary rejects;
- dropping an invalid secondary is not treated as preserving the original
  diagnostic.

### D5 -- Integration examples and manifest

Add compact examples that read as catalog literature:

- a successful Boolean parse;
- a malformed Boolean expression with a located parse error;
- a formatter-idempotence example;
- a diagnostic with a primary and one secondary span;
- an offset-preserving view example aligned with the `.ken.md` seam, without
  implementing `.ken.md` extraction.

The manifest must state:

- derivation from built-ins and earlier packages;
- expected `trusted_base()` delta is zero;
- laws proved now;
- richer deferred work: full Ken syntax reflection, compiler parser API,
  source maps, CommonMark, nested fences, LSP/editor rendering.

## 5. Hard Gates

- **G1 -- Zero trust delta.** No kernel, primitive registry, compiler parser,
  CLI loader, export/provenance, or `Cargo.lock` change. No `Axiom` in
  `catalog/packages/parsing/`. `trusted_base()` deltas for public definitions and laws
  are empty.
- **G2 -- Byte artifact identity.** Source/span tests prove offsets are over
  `sourceBytes`, not normalized `String` or copied substrings.
- **G3 -- Valid spans everywhere.** Located syntax, parse success, parse
  failure, diagnostics, and related spans all carry `ValidSpan` obligations.
- **G4 -- Total parser result.** Parse calls return `Parsed` or `Failed`; no
  crash, loop, unstructured string failure, or unlocated failure.
- **G5 -- Small grammar only.** The accepted Boolean grammar is exactly the v1
  fully parenthesized grammar. No compiler AST public API and no claim to parse
  Ken source.
- **G6 -- Round trip and formatter laws.** Parser/printer round trip holds
  modulo `eraseSpans`; formatter preservation and idempotence hold over bytes.
- **G7 -- Diagnostic validity flips.** Out-of-range primary and out-of-range
  secondary diagnostics reject. Invalid secondaries are not silently discarded.
- **G8 -- `.ken.md` boundary.** The package treats `.ken.md` compiled buffers as
  offset-preserving derived views only. It does not change or reimplement
  `ken-md-literate`.
- **G9 -- Workspace green.** Focused package tests pass, relevant conformance
  seed expectations are exercised, and the normal workspace gate is green.

## 6. Review Path

Language implementer builds in D-slices and hands exact heads to Language
leader/QA. Language QA re-derives G1-G9 from the diff and runs focused tests.

Architect re-certifies the built diff for:

- byte-artifact source/span identity;
- parser/printer and formatter law shape;
- diagnostic primary+secondary validity;
- `.ken.md` derived-view boundary;
- zero trust delta.

Conformance-validator reviews only if the build changes conformance. The
expected build should not edit `conformance/`; it should make the existing
red-until-built CAT-5 seed green through package/test implementation.

Integrator merge uses the normal build/test, conformance, clean-room
provenance, and path-guard gates. After merge, Language posts implementer, QA,
and leader retros before Steward closes CAT-5 build.
